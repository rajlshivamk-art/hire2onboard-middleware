2. Candidate Application Details (State Diagram)

stateDiagram-v2
    [*] --> Applied
    Applied --> Screening : Recruiter Reviews
    Screening --> Round1 : Pass Screening
    Screening --> Rejected : Fail Screening

    Round1 --> Round2 : Pass Technical
    Round1 --> Rejected : Fail Technical

    Round2 --> Round3 : Pass Deep Dive
    Round2 --> Rejected : Fail Deep Dive

    Round3 --> Management : Pass System Design
    Round3 --> Rejected : Fail System Design

    Management --> Offer : Hiring Manager Approval
    Management --> Rejected : No Fit

    state Offer {
        [*] --> DraftOffer
        DraftOffer --> SentOffer : Generate & Email
        SentOffer --> Accepted : Candidate Signs
        SentOffer --> Declined : Candidate Rejects
    }

    Accepted --> Onboarding : Move to Onboarding
    Onboarding --> Hired : All Tasks Complete
    
    Hired --> [*]
    Rejected --> [*]
    Declined --> [*]

3. System Architecture (C4 Container)

C4Context
    title System Context Diagram for Recruitment Software

    Person(candidate, "Candidate", "An applicant looking for a job")
    Person(recruiter, "Recruiter", "Manages job postings and candidates")
    Person(interviewer, "Interviewer", "Conducts interviews and provides feedback")
    Person(manager, "Hiring Manager", "Makes final hiring decisions")

    System_Boundary(ats, "Recruitment Tracking System") {
        Container(web_app, "Web Application", "React, Vite, TypeScript", "Provides the UI for all users to interact with the system.")
        Container(api, "API Gateway", "Python, FastAPI", "Handles business logic and data processing.")
        ContainerDb(database, "Database", "MongoDB", "Stores users, jobs, applications, and metadata.")
        ContainerDb(file_storage, "File Storage", "Local / S3", "Centralized document storage.")
    }

    Rel(candidate, web_app, "Applies for jobs via", "HTTPS")
    Rel(recruiter, web_app, "Manages pipeline via", "HTTPS")
    Rel(interviewer, web_app, "Submits feedback via", "HTTPS")
    Rel(manager, web_app, "Approves offers via", "HTTPS")

    Rel(web_app, api, "API Calls", "JSON/HTTPS")
    Rel(api, database, "Reads/Writes Data", "Mongo Driver")
    Rel(api, file_storage, "Uploads/Retrieves Files", "File System/SDK")
